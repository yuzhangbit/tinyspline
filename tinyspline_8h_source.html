888 /******************************************************************************
889 * *
890 * :: Transformation functions *
891 * *
892 * TinySpline is a library focusing on transformations. That is, most *
893 * functions are used to transform splines by modifying their state, e.g., *
894 * their number of control points, their degree, and so on. Accordingly, each *
895 * transformation functions specifies an input and output parameter (along *
896 * with the other parameters required to calculate the actual transformation). *
897 * By passing a different pointer to the output parameter, the transformation *
898 * result is calculated and stored without changing the state of the input. *
899 * This is in particular useful when dealing with errors as the original state *
900 * will never be modified. For instance, let's have a look at the following *
901 * code snippet: *
902 * *
903 * tsBSpline in = ... // an arbitrary spline *
904 * tsBSpline out; // result of transformation *
905 * *
906 * // Subdivide 'in' into sequence of bezier curves and store the result *
907 * // in 'out'. Does not change 'in' in any way. *
908 * tsError err = ts_bspline_to_beziers(&in, &out); *
909 * if (err != TS_SUCCESS) { *
910 * // fortunately, 'in' has not been changed *
911 * } *
912 * *
913 * Even if 'ts_bspline_to_beziers' fails, the state of 'in' has not been *
914 * changed allowing you to handle the error properly. *
915 * *
916 * Unless stated otherwise, the order of the parameters for transformation *
917 * functions is: *
918 * *
919 * function(input, [additional_input], output, [additional_output]) *
920 * *
921 * 'additional_input' are parameters required to calculate the actual *
922 * transformation. 'additional_output' are parameters storing further result. *
923 * *
924 * Note: None of TinySpline's transformation functions frees the memory of the *
925 * output parameter. Thus, when using the same output parameter multiple *
926 * times, make sure to free memory before each call. Otherwise, you will *
927 * have a bad time with memory leaks: *
928 * *
929 * tsBSpline in = ... // an arbitrary spline *
930 * tsBSpline out; // result of transformations *
931 * *
932 * ts_bspline_to_beziers(&in, &out); // first transformation *
933 * ... // some code *
934 * ts_bspline_free(&out); // avoid memory leak. *
935 * ts_bspline_buckle(&in, &out); // next transformation *
936 * *
937 * If you want to modify your input directly without having a separate output, *
938 * pass it as input and output at once: *
939 * *
940 * tsBSpline s = ... // an arbitrary spline *
941 * tsReal *knots = ... // a knot vector *
942 * *
943 * ts_bspline_set_knots(&s, knots, &s); // copy 'knots' into 's' *
944 * *
945 * Note: If a transformation function fails *and* input != output, all fields *
946 * of the output parameter are set to 0/NULL. If input == output, your *
947 * input may have an invalid state in case of errors. *
948 * *
949 ******************************************************************************/
